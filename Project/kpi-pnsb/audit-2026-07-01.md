# PNSB BSC KPI — System / Flow / DB Audit (2026-07-01)

Whole-system audit of `kpi_pnsb` (Laravel 13 + Livewire 4 + MySQL) on `dev`, run as 3 parallel read-only audits (authorization/security, flow integrity, DB design). Builds on the prior P0–P3 review — those fixes were **re-verified intact**. The two HIGH security items were personally re-verified in code.

**Status legend:** ✅ FIXED (remediated 2026-07-01, see decision-log) · ⬜ OPEN.

## CRITICAL
- **C1 ✅ — Scorecard regeneration crashed after a soft-delete.** `unique(user_id, cycle_id)` counts trashed rows; `Cycles/Show::generateScorecards` `firstOrCreate` ignores trashed → INSERT collides → SQLSTATE 23000. Once any card was soft-deleted, regenerating that cycle threw for that employee. **Fix:** `withTrashed()->firstOrNew` + `restore()` (app-level). *NB the "add deleted_at to the unique key" trick fails on MySQL — MySQL treats NULLs as distinct, so it wouldn't block two live cards, and MySQL refuses to drop an index a FK needs.*

## HIGH
- **H1 ✅ [Security, verified] — hr-admin could delete/deactivate a super-admin from the user LIST.** `Users/Index::delete()`/`toggleActive()`/`confirmDelete()` guarded only on "not myself" (every other user mutation uses `canManage()`). **Fix:** add `canManage()` guards + hide controls for unmanageable rows.
- **H2 ✅ [Security, verified] — competency evidence world-readable + any file type.** Media on the `public` disk, linked by enumerable `/storage` URL, no retrieval authz (IDOR); uploads had no MIME limit (`.html/.svg` = stored XSS). **Fix:** private `local` disk + authorised `CompetencyAttachmentController` (`Scorecard::isViewableBy` = owner/appraiser/admin) + MIME allowlist.
- **H3 ✅ [Flow] — MOD1/MOD2 had no ordering lock.** Both rounds open all through moderation → a per-dept HR re-band could silently override the ED's org-wide MOD2 (summary derives MOD2 from current grade). **Fix (chosen: explicit Seal-MOD1 gate):** `AppraisalCycle.mod1_sealed_at/by`; HR seals → MOD1 read-only, MOD2 opens; lock requires seal (seal→MOD2→lock).
- **H4 ✅ [Flow] — `manager_id` snapshot never re-synced.** Set only at `AddEmployee` = head at add-time; changing a dept head left employees pointing at the old head → wrong appraiser or permanent stall (which then blocks the whole cycle at moderation). **Fix (chosen: data-consistency only):** `Departments/Edit` re-syncs subordinates on head change; head-of-own-dept no longer self-manages; generation warning widened to missing/self/inactive appraiser.
- **H5 ✅ [DB] — `users` had no foreign keys.** `department_id`/`designation_id`/`manager_id` were plain integers → dangling pointers, silently broken chain. **Fix:** added the 3 FKs (all `nullOnDelete`) in a later migration.

## MEDIUM (⬜ open — parked)
- **M1 [Flow]** — advancing to `moderation` is all-or-nothing; no exempt/void path, so one no-show can deadlock the cycle (`Cycles/Show::scorecardsReadyForPhase`). *(H4 reduces the trigger; the escape hatch itself is still open.)*
- **M2 [Security]** — `DatabaseSeeder` runs `UserSeeder` (super-admin, password `password`) + `KpiDemoSeeder` **unconditionally**; if run in prod → super-admin with a known password. Gate demo seeding behind `local`.
- **M3 [Flow]** — lifecycle transitions (`approveKpis`/`completeReview`/`Results::finalize`/`Show::submit`) not re-checked under `lockForUpdate`; double-submit dupes status/override audit rows (scores idempotent).
- **M4 [DB]** — `scorecard_kpis` lacks `unique(scorecard_id, kpi_id)` (its competency twin has one); concurrent generation could duplicate KPI lines → weights ≠ 100.
- **M5 [Flow]** — Reports/Dashboard pools use `whereNotNull('grade')` only, not `MODERATION_STATUSES`; fine today, fragile if a reopen path is ever added. `User::performanceHistory` already does it right.
- **M6 [DB]** — soft-delete vs unique on `users.email`/manual `employee_no`, `review_periods(cycle_id,sequence)`, `bell_curve_targets(cycle_id,grade)`, `scorecard_templates(category)` — a trashed row reserves the value → raw integrity error on re-create/re-seed.
- **M7 [DB]** — force-deleting a scorecard/review-period cascades away moderation_logs/score_overrides/status_logs/period-scores (append-only trails). Consider `restrictOnDelete`.
- **M8 [Flow]** — head added to own dept self-appraised. *(Fixed as part of H4.)*

## LOW (⬜ open)
- L1 unvalidated grade string in `reband` (moderator-only, integrity not escalation) · L2 competency `description` read live not snapshot · L3 `Users/Show` read lacks `canManage` (info disclosure; intentionally NOT fixed — broke behaviour/tests, out of scope) · L4 demo seeders not env-gated · L5 audit excludes only `password` (not `remember_token`) · L6 `final_score` assumes template weights sum 100 · L7 perspective name not snapshotted on `scorecard_kpis` · L8 no `(cycle_id,status)`/`(cycle_id,grade)` composite indexes · L9 employee can edit results during appraisal phase (by design).

## Verified well-designed
Enum↔constant alignment (no drift); KPI snapshot architecture correctly consumed; Scorecards/Appraisals/Moderation/Reports IDOR-safe; routes fully behind auth/verified; audit search parameterized; dev helpers env-gated; no backward status transitions; moderation preserves final_score; competency-zero-lines blocked; kpi_score withheld until all periods scored.
